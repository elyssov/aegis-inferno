# Коммуникатор "ФАКЕЛ" — Техническое задание на интерфейс

---

## 1. Концепция

Интерфейс квантового коммуникатора "Факел-12" — военный полевой терминал спецподразделения по паранормальным исследованиям. Визуально: скрещение советского военного прибора с футуристическим квантовым устройством. Тёмный, зелёный, тревожный. По мере нарастания аномалии — покрывается помехами, глитчами, искажениями.

Устройство используется в двух сценариях:

**Сценарий 1 (пролог):** "Факел" найден случайно. Интерфейс проще — нет панелей здоровья отряда. Один канал связи с протагонистом. Индикаторы: заряд батареи (99%), сила сигнала, статус соединения.

**Сценарий 2 (основная игра):** Штатный "Факел-12" офицера ОПИ. Полный интерфейс: панели здоровья отряда (4 оперативника), командный лог, кнопки действий, предиктивные заготовки, индикатор искажения реальности.

---

## 2. Структура экрана (сценарий 2 — полный)

Экран разделён на вертикальные зоны сверху вниз:

### 2.1. Верхняя панель статуса (фиксированная)

Одна строка, мелкий моноширинный шрифт.

```
СОЕДИНЕНИЕ: СТАБИЛЕН  |  ЗАРЯД: 85%  |  ИСКАЖЕНИЕ: 7%  |  ВРЕМЯ МИССИИ: 01:47:33
```

- **СОЕДИНЕНИЕ** — статус квантового канала. Значения: СТАБИЛЕН (зелёный), НЕСТАБИЛЕН (жёлтый, мигает), КРИТИЧЕСКИЙ (красный, мигает быстро), ПОТЕРЯН (красный, статичный).
- **ЗАРЯД** — батарея "Факела". Начинается с 99%. Уменьшается очень медленно (косметика, не влияет на геймплей). Зелёный >50%, жёлтый 20-50%, красный <20%.
- **ИСКАЖЕНИЕ** — ключевой показатель. Процент искривления реальности вокруг протагониста. Начинается с 0%. Растёт по ходу сюжета. Влияет на визуальные эффекты интерфейса (помехи, глитчи, шум). При >80% — интерфейс едва читаем.
- **ВРЕМЯ МИССИИ** — реальный таймер от начала игры. В сценарии 2 также служит напоминанием о дедлайне ритуала.

### 2.2. Панель отряда (только сценарий 2)

Четыре горизонтальные плашки, каждая содержит:

```
┌─────────────────────────────────────────┐
│ СИМОНОВ          ♥ ████████░░  82%      │
│ Сейф | Оперативник         [АКТИВЕН]   │
└─────────────────────────────────────────┘
```

- **Имя** — фамилия оперативника. Цвет уникальный для каждого (зелёный, голубой, оранжевый, фиолетовый).
- **Кардиограмма/полоска здоровья** — анимированная линия ЭКГ пока жив. Флэтлайн при гибели.
- **Процент** — состояние здоровья.
- **Класс** — "Сейф", "Псионик", "Капеллан".
- **Статус** — [АКТИВЕН] зелёный, [РАНЕН] жёлтый, [КРИТИЧЕН] красный мигающий, [ПОТЕРЯН] серый с флэтлайном.

При гибели оперативника: кардиограмма переходит во флэтлайн (анимация замедления пульса → прямая линия). Плашка темнеет, становится полупрозрачной. Имя зачёркивается.

### 2.3. Основное текстовое поле (лог сообщений)

Занимает основную часть экрана. Прокручиваемый лог сообщений снизу вверх (новые внизу).

Каждое сообщение:

```
[09:37] ГРОМОВ: Мы держимся. Связь прерывается...
```

- **Таймстемп** — серый, мелкий.
- **Имя отправителя** — цвет соответствует персонажу. ЦЕНТР (сообщения игрока) — белый.
- **Текст** — основной цвет: зеленовато-белый (#c8e6c9).
- **Системные сообщения** — жёлтый курсив, без имени: `--- Переключение на вспомогательный канал...`
- **Сообщения от враждебных сущностей** — красный, искажённый шрифт, с глитч-эффектом: `Съ'артототех: К'РАХ-ХА! ЩИ-ПТА-ТАХ!`
- **Сообщения Радмилы** — тёплый золотистый (#daa520), с вертикальной полоской-акцентом слева.

Эффект "печатной машинки" — сообщения протагониста появляются посимвольно, с задержкой, имитируя голосовой ввод через "Факел". Системные — мгновенно.

**Эффект деградации текста:** при высоком искажении (>50%) сообщения начинают содержать артефакты:
- Случайные символы заменяются на мусор: "Я вижу лагерь впер░░и"
- Буквы "плывут" — CSS transform с лёгким случайным смещением
- Периодические "вспышки" строк — строка на долю секунды заменяется случайным мусором и возвращается

### 2.4. Разделительная линия с индикатором

Тонкая горизонтальная линия между логом и кнопками. По ней бежит зелёная точка слева направо — индикатор активности канала. При нестабильном соединении — точка мерцает, двигается рывками.

### 2.5. Панель ввода

**Поле текстового ввода** — для свободного текста игрока. Стиль: тёмный фон, зелёная рамка, моноширинный шрифт. Placeholder: `> Введите сообщение...`

**Кнопки быстрых действий** — две основные + одна специальная:

Основные кнопки (контекстно меняются в зависимости от ситуации):
```
[ВАРИАНТ А]        [ВАРИАНТ Б]
```
Текст на кнопках меняется по ситуации: "Беги!" / "Бей его!", "Да" / "Нет", "Иди вперёд" / "Обойди", и т.д.

Специальная кнопка:
```
[ОСОБЫЕ ДЕЙСТВИЯ]
```
При нажатии — раскрывается панель с контекстными кнопками:
- Для офицера ОПИ: "Запросить аналитику", "Сверить с базой данных", "Отправить координаты"
- Для координатора пролога: "Описать что вижу", "Проверить карту"

### 2.6. Нижняя панель предиктивных заготовок

Горизонтальный скролл кнопок-подсказок (как предиктивный ввод в мессенджерах):

```
[Доложите обстановку]  [Отступайте!]  [Держитесь!]  [Что вы видите?]
```

Заготовки меняются контекстно. В бою — боевые. В диалоге — дипломатические. При панике протагониста — ободряющие.

## 3. Визуальный стиль

### 3.1. Цветовая палитра

```
Фон:                #0a0f0a (почти чёрный с зеленоватым оттенком)
Фон панелей:         #0d1a0d (чуть светлее, тёмно-зелёный)
Рамки:               #1a3a1a (тёмно-зелёный)
Основной текст:      #c8e6c9 (светло-зелёный)
Акцентный зелёный:   #00ff41 (яркий терминальный зелёный)
Предупреждение:      #ffa000 (янтарный)
Опасность:           #ff1744 (красный)
Системный:           #607d60 (приглушённый зелёный)
Радмила:             #daa520 (золотистый)
Враждебный:          #ff1744 (красный с глитч-эффектом)
```

### 3.2. Типографика

- Основной шрифт: `'Share Tech Mono'` или `'IBM Plex Mono'` (Google Fonts) — моноширинный, военный.
- Заголовки/статус: `'Orbitron'` или `'Rajdhani'` — футуристический, угловатый.
- Враждебные сообщения: основной шрифт + CSS искажения (skew, scale, opacity мерцание).

### 3.3. Текстуры и эффекты

**Шум (постоянный):** лёгкий CSS grain-overlay поверх всего интерфейса. Opacity 3-5% в нормальном состоянии, до 15-20% при высоком искажении.

**Scan lines:** горизонтальные полосы поверх экрана (repeating-linear-gradient), opacity 5%. Эффект старого монитора.

**Виньетка:** затемнение по краям экрана (radial-gradient).

**Глитч-эффект (при искажении):**
- Горизонтальные полосы смещения (clip-path + transform: translateX)
- Цветовое расщепление (text-shadow с красным и синим смещением)
- Периодические "вспышки" белого
- Мерцание отдельных элементов

**Пульсация рамок:** тонкие зелёные рамки элементов периодически "пульсируют" — плавное увеличение и уменьшение яркости.

### 3.4. Звуковые маркеры (заглушки для альфы)

```
- incoming_message: тихий "блип"
- system_alert: двойной "бип"
- flatline: непрерывный тон
- distortion_increase: низкочастотный гул
- hostile_message: искажённый скрежет
- button_click: щелчок тумблера
```

## 4. Адаптивность

### 4.1. Mobile-first

Основной формат — мобильный телефон (вертикальная ориентация). Интерфейс "Факела" должен выглядеть как реальное приложение на телефоне игрока. Это ключевой элемент иммерсии.

### 4.2. Desktop

На десктопе — центрированный контейнер шириной ~420px (пропорции смартфона), с тёмным фоном вокруг. Опционально: рамка "корпуса устройства" вокруг контейнера.

## 5. Динамические состояния интерфейса

### 5.1. Нормальное (искажение 0-20%)

Чистый интерфейс. Лёгкий шум. Scan lines едва видны. Все элементы читаемы. Зелёная цветовая схема.

### 5.2. Тревожное (искажение 20-50%)

Шум усиливается. Периодические горизонтальные полосы помех (раз в 10-15 секунд). Рамки иногда мерцают. Текст иногда дёргается. Цвет акцента уходит в желтоватый.

### 5.3. Опасное (искажение 50-80%)

Сильный шум. Частые помехи. Глитч-эффекты каждые 3-5 секунд. Текст периодически "разъезжается". Отдельные символы заменяются мусором. Кардиограммы дёргаются. Рамки мерцают красным.

### 5.4. Критическое (искажение >80%)

Экран едва читаем. Постоянные помехи. Цветовое расщепление на всём тексте. Интерфейс "трясётся". Scan lines жирные, грубые. Периодически экран "гаснет" на долю секунды. Сообщения от враждебных сущностей занимают экран поверх интерфейса.

---

## 6. Особые экранные события

### 6.1. Гибель оперативника

Кардиограмма замедляется → флэтлайн. Плашка мигает красным 3 раза. Затем темнеет. Имя перечёркивается. Звук непрерывного тона (1 секунда). Системное сообщение: `--- ПОТЕРЯ СИГНАЛА: СИМОНОВ ---`

### 6.2. Появление враждебной сущности в канале

Экран на секунду покрывается красными символами. Затем — сообщение красным искажённым шрифтом. Рамки интерфейса мигают красным. Шум резко усиливается на 2-3 секунды.

### 6.3. Потеря связи

Экран уходит в "снег" (белый шум). Надпись по центру: `ВОССТАНОВЛЕНИЕ КВАНТОВОГО ТОННЕЛЯ — ОЖИДАЙТЕ`. Индикатор прогресса. Длительность: сюжетно обусловлена (от 30 секунд до нескольких минут реального времени).

### 6.4. Сообщение от Радмилы (пролог)

Системное: `--- Перехвачен вспомогательный канал ---`. Затем сообщения золотистым цветом. Вертикальная полоска-акцент слева — тёплый золотой (#daa520). Шум на секунду УМЕНЬШАЕТСЯ (контраст — в её присутствии спокойнее).

### 6.5. Ядерный удар / катастрофическое событие

Экран заливает белым на 0.5 секунды. Затем — красный импульс. Все показатели скачут. Интерфейс "перезагружается" (анимация boot sequence 2-3 секунды).

## 7. Интеграция с игровым движком

### 7.1. Сценарный движок

State machine. Состояния: чекпоинты сюжета. Переходы: по выбору игрока (кнопки) или по таймеру.

```
state: {
  chapter: 'prologue',        // 'prologue' | 'main'
  checkpoint: 'cave_escape',   // текущий чекпоинт
  distortion: 12,              // процент искажения (0-100)
  battery: 85,                 // заряд (косметика)
  protag_class: null,          // 'soldier' | 'psion' | 'chaplain' (сценарий 2)
  team_alive: ['simonov','starkov','zvyagintsev','gromov'],  // живые
  inventory: ['knife','fakel'],
  flags: {
    radmila_romance: false,
    priest_alive: false,
    hunters_met: false,
    child_demon_trusted: false,
    // ...
  },
  timer_seconds_remaining: null  // null если нет дедлайна
}
```

### 7.2. AI-генерация реплик

Claude API на бэкенде. System prompt содержит: текущее состояние, личность протагониста, контекст сцены, допустимые ветвления. Claude генерирует реплики протагониста и NPC в рамках сценарного скелета.

Ограничение: AI НЕ решает за игрока. AI генерирует реплики NPC и описания. Выбор всегда за игроком (кнопки или свободный текст).

## 8. Приоритеты реализации

### Альфа (proof of concept):
- Интерфейс "Факела" (HTML/CSS/JS).
- Статичная демо-сцена: 3 минуты пролога (от первого контакта до побега из пещеры).
- Эффекты: шум, scan lines, печатная машинка, базовый глитч.
- Кнопки выбора (2 варианта).
- Без AI — заскриптованные реплики.

### Бета:
- Полный пролог (Карелия).
- Интеграция Claude API для генерации реплик.
- Все визуальные эффекты (4 уровня искажения).
- Звуковые заглушки.
- Mobile + Desktop адаптивность.

### Полная версия:
- Основная кампания (Скобелевск).
- Выбор протагониста.
- Панель отряда с кардиограммами.
- Таймер ритуала.
- Полное звуковое оформление.
- Множественные концовки.
